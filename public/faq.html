Frequently Asked Questions

Everything you need to know about IP addresses and geolocation.

What is my IP address?

Your IP (Internet Protocol) address is a number assigned to your connection by your ISP. Visit the IPScope homepage to see your public IPv4 and IPv6 addresses, location, and ISP details automatically.

What is the difference between IPv4 and IPv6?

IPv4 uses 32-bit addresses (e.g. 203.0.113.45). IPv6 uses 128-bit addresses (e.g. 2001:db8::1) to solve IPv4 exhaustion. Many networks have both; IPScope detects each when available.

How accurate is IP geolocation?

Typically city or region level (10–50 km radius). It reflects where your ISP registers the IP block, not your GPS location. VPNs, proxies, and corporate networks often show a different city or country.

Can websites see my exact home address from my IP?

No. Standard IP geolocation cannot determine your street address. It resolves to approximate city-level coordinates. See our Disclaimer.

Does IPScope use third-party geolocation APIs?

No API keys required. Lookups use offline DB-IP and GeoLite2 MMDB files on the server. RDAP queries use public registry endpoints for ASN and network ownership data.

Can IPScope detect VPN or proxy usage?

IPScope shows heuristic indicators (datacenter ISP names, proxy chain headers, known VPN ASNs). This is indicative, not definitive proof of VPN use.

How do I look up someone else's IP?

Enter any public IPv4 or IPv6 address in the search box on the homepage, or visit /lookup/1.2.3.4 directly. Only use this for lawful purposes.

Is there an API?

Yes. See API documentation for /api/me, /api/lookup/:ip, and bulk lookup endpoints.

How do I self-host IPScope?

Clone the GitHub repo, run npm install, npm run download-db, then npm start. The server runs on port 3920 by default.

Why does my location show the wrong city?

Your ISP may route traffic through a hub in another city, you may be on a VPN, or the database record may be outdated. Mobile networks often show the carrier's gateway city.
